## Code Changes

### Terminal backend and protocol

- `apps/gitboard/src/api/terminal/provider-registry.ts` now exposes a real `pty` provider backed by a Node `node-pty` helper and a readonly `specialist-feed` provider backed by `specialists feed <job_id> --follow`.
- `apps/gitboard/src/api/terminal/node-pty-helper.cjs` loads native `node-pty` under Node and communicates output/input/resize/exit over JSON-lines to the Bun server.
- `apps/gitboard/src/api/terminal/bridge.ts` now supports provider `jobId` forwarding, pending upgrade connection ids, per-connection auth context, secure reattach tokens, owner-only input/resize/exit, and stale disconnect cleanup.
- `packages/core/src/terminal/protocol.ts` now allows optional `jobId` in `open` payloads and validates it as a string when present.
- `apps/gitboard/src/api/server.ts` routes terminal WS upgrades through origin/admin checks before upgrade and uses `TerminalBridge.handleUpgrade` to avoid pre-socket message races.
- `apps/gitboard/src/core/shell-provider-policy.ts` now supports local origin checks, request-bound shell admin token checks, local dev host recognition, default zsh allowlisting, and safer env scrubbing defaults.

### Terminal frontend

- `apps/gitboard/src/dashboard/components/shell/TerminalTabPanel.tsx` sends full protocol envelopes, handles open/attach/reconnect, clears stale sessions on `not_found`, shows toolbar/status/error UI, and preserves terminal output while active.
- `apps/gitboard/src/dashboard/components/terminal/TerminalStream.tsx` imports xterm CSS, creates xterm once per mount, keeps callbacks fresh via refs, guards focus/fit, refits after font load, and uses terminal-safe font metrics.
- `apps/gitboard/src/dashboard/styles/globals.css` adds terminal panel/stream styles, status toolbar, xterm font/kerning controls, and `#141414` terminal background.
- `apps/gitboard/vite.config.ts` adds explicit WebSocket proxying for `/api/console/terminal/ws`.
