**Arc 2 (post-close hardening, after `/session-close-report` was first written):** operator reported `/console/graph` stuck on "Loading graph…". Diagnosis surfaced two compounding bugs the morning's PR #12 didn't catch: (a) `DoltClient.getIssues` had an N+1 (2 sub-queries per row × `limit=1000` ≈ 2k MySQL round-trips per request on large repos like `unitAI`), and (b) `apps/gitboard/src/api/routes/specialists.ts` recreates the attach-pool every 2 seconds, which reset the dead-cache from PR #12 between every recreation — letting the observability hot-loop re-establish itself at 90%+ CPU. Then operator reported the cockpit "Last output" was always empty — root-caused to `observability/dao.ts` never including `last_output` in its SELECT, so the column never reached the API. Shipped fix in PR #15 with PRAGMA-probed graceful fallback for older schemas and 2 new round-trip tests. Closed `forge-ujwm`.

**PR #14 — `fix/dolt-client-batch-hydration` → `main` (`d209d5a`):**
