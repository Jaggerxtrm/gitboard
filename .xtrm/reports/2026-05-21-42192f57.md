# Session Report — 2026-05-21

## Summary

Single-orchestrator session focused on Gitboard dashboard high-availability and realtime continuity. Started from a verification audit confirming the residual UI latency was frontend orchestration + sync work on the request path, not the DB layer. Planned 4 P0/P1 beads under epic forge-vi4j (WS reconnect, GitHub HA store-merge, graph/specialists serve-stale + async Dolt, instrumentation), shipped all four through executor → reviewer chains, then ran dev locally and verified live. Live probing surfaced three sibling regressions that forge-5i43's acceptance criteria did not name explicitly — `listRepos()` sync FS walk on every specialists request, `createAttachPool()` sync ATTACH inside a microtask, and per-repo dao queries blocking the event loop in background — all filed as `discovered-from` follow-ups and shipped same session. Two unrelated bugs surfaced during live testing (observability DB schema_version=0 false reject, graph project UUID lookup returning fallback) were also filed and fixed. Closed with chip priority-rail UI polish so connection arrows no longer overlap the rail. The recurring pattern — "spirit-of-AC" gaps where reviewers enforced named gates but missed equivalent sibling code paths — was captured as a durable memory: **bead AC for performance work should specify behaviors, not named implementations**. Net result: cold parallel API calls dropped from ~1.9s to <50ms, warm graph holds under 300ms in all observed waves, observability DB now attaches correctly, graph empty-state correctly distinguishes loading from truly-empty, and visible HA layer (DB-first, WS freshness, REST replenishment) is now structurally sound across graph/specialists/beads/github.

## Code Changes

### Server / API

- **HA / serve-stale layer**: `apps/gitboard/src/core/graph-dao.ts`, `apps/gitboard/src/api/routes/{graph,specialists,beads}.ts` — replaced TTL-keyed cache with epoch-keyed cache identity, added `getGraphSnapshot()` sync return with `freshness` metadata, moved Dolt warm to `void warmGraphState()` background task.
- **Async ATTACH pool**: `apps/gitboard/src/server/observability/attach-pool.ts` — `warmAttachPool` chunks ATTACH operations 5-at-a-time with `yieldToEventLoop()` between batches; `withAttached` no longer triggers synchronous attach loop.
- **Chunked dao queries**: `apps/gitboard/src/server/observability/dao.ts` — `readJobsChunked` yields per-repo so `inFlightJobs`/`recentJobs` don't block the event loop.
- **listRepos cache**: `apps/gitboard/src/server/observability/registry.ts` — module-level cache with short refresh window.
- **Schema gate**: `apps/gitboard/src/server/observability/schema-guard.ts` (new) — table-shape compatibility check replaces version-only gate; `attach-pool.ts` fixed log-string bug.
- **resolveProject**: `apps/gitboard/src/core/graph-dao.ts` — matches by id/name/path with proper fallback.
- **Instrumentation**: `apps/gitboard/src/api/routes/github.ts` (`t_db`/`t_serialize`), `apps/gitboard/src/api/server.ts`, `apps/beadboard/src/api/routes/beads.ts`.

### Dashboard / Client

- **WS resume**: `apps/gitboard/src/dashboard/lib/ws.ts` — `unsubscribe()` no longer deletes `lastSeqByChannel`; added `isWsDebugEnabled`/`wsDebugLog` for routine seq/event logs.
- **Store-merge semantics**: `apps/gitboard/src/dashboard/stores/github.ts` — `setX(empty)` no-ops against non-empty state; explicit `clearX` actions added.
- **Per-tab cache**: `apps/gitboard/src/dashboard/components/shell/MainPane.tsx` — `useGithubRepoData` reads cached store rows immediately; performance marks for `tab_switch:*:start/:rendered`.
- **Freshness UX**: `apps/gitboard/src/dashboard/hooks/useGraphData.ts`, `apps/gitboard/src/dashboard/pages/console/Graph.tsx` — distinct empty states for fresh/stale/degraded; one auto-refetch on stale.
- **Sync-hint preserve**: `apps/gitboard/src/dashboard/hooks/useGithubActivity.ts` — `preserveVisibleState` mode for background refresh.
- **Chip rail**: `apps/gitboard/src/dashboard/styles/globals.css` — `.g-node` priority rail moved to `::before` pseudo so connection arrows touch a clean 1px outer border.
